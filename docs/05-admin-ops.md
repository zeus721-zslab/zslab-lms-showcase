# 운영자 도구

## 개요

LMS 초기 구조에서 운영자가 학습자 1명의 정보를 확인하려면 회원 관리·수강 관리·주문 관리·자격증 관리 등 여러 메뉴를 순차적으로 탐색해야 했습니다. 강좌 운영 역시 강좌·시험·분반 메뉴가 분리되어 있어 단일 강좌를 운영하는 동선이 복잡했습니다. 학기가 반복될 때마다 분반·강좌 구성을 수동 재입력해야 했고, 글로벌 검색이 정적 페이지 이동 전용으로만 동작해 학습자·주문을 이름으로 빠르게 찾을 수단이 없었습니다.

이를 해소하기 위해 4개 운영 도구를 신규 구현·통합했습니다.

---

## 🛠 핵심 도구

### 1. 학습자 360 뷰

**무엇**: `/lms-manage/users/[id]` 회원 상세 페이지에 6탭 구조를 추가했습니다. 기존 회원 관리(역할·정지·수강 등록) 기능을 완전히 보존하면서 학습자 1명의 모든 데이터를 한 화면에서 조회합니다.

**왜**: 콜 응대 시나리오에서 학습자 정보를 확인하려면 회원·수강·주문·자격증·문의 메뉴를 각각 열어야 했습니다. 화면 전환이 많아 응대 품질에 직접 영향을 줬습니다.

**결과**: 단일 진입점에서 KPI 9종(가입일·최근 로그인·수강 중·수료·총 학습시간·결제액·환불액·자격증 수·미처리 문의)과 6개 데이터 영역을 즉시 조회·관리합니다.

| 탭 | 내용 |
|---|---|
| 개요 | KPI 9종 요약 위젯 |
| 수강 | 수강 이력 테이블 · 강좌 상세 이동 |
| 시험·과제 | 시험 / 과제 토글 · 채점 상세 이동 |
| 결제 | 결제 이력 · 금액·상태 |
| 자격증 | 발급 이력 · 시리얼 번호 |
| 문의 | 1:1 문의 목록 · 미읽음 강조 |

---

### 2. 강좌 운영 허브

**무엇**: `/lms-manage/courses/[id]` 강좌 상세 페이지를 7탭 구조로 확장했습니다. 기존 정보·차시·문항 탭을 보존하고 수강생·시험과제·스태프·통계 탭을 추가했습니다.

**왜**: 강좌에서 시험 관리·수강생 조회·스태프 배정으로 이동하는 동선이 각기 다른 메뉴에 분산되어 있었습니다. 강좌 운영 사이클 전체가 한 화면에 모이지 않았습니다.

**결과**: 강좌 1개를 7탭에서 종합 관리하며, 수강생 탭에서 행을 클릭하면 학습자 360 뷰로 직접 연결됩니다. 통계 탭에서 수강 현황·매출·평가 KPI를 한 화면에서 확인합니다.

| 탭 | 내용 |
|---|---|
| 정보 | 강좌 메타데이터 (기존 보존) |
| 차시 | 차시 빌더 (기존 보존) |
| 문항 | 문제은행 연결 (기존 보존) |
| 수강생 | 수강생 목록 · 학습자 360 뷰 연계 |
| 시험·과제 | 시험 / 과제 토글 · 상세 이동 |
| 스태프 | 스태프 배정·해제 |
| 통계 | 수강 현황·매출·평가 KPI |

---

### 3. 글로벌 검색 (Command Palette)

**무엇**: Ctrl+K로 열리는 Command Palette에 실시간 도메인 검색을 추가했습니다. 학습자(이름·이메일)·강좌(제목)·시험(제목)·주문(주문번호)을 키워드로 검색하고, 결과 클릭 시 해당 상세 페이지로 이동합니다. 기존 정적 페이지 이동 기능은 그대로 보존됩니다.

**왜**: 기존 Command Palette는 메뉴 이동 전용이었습니다. 학습자·주문을 이름·번호로 찾으려면 각 목록 페이지에서 개별 검색해야 했습니다.

**결과**: 키워드 2자 이상 입력 시 300ms 디바운스 후 4개 도메인을 동시 검색합니다. 결과는 도메인별로 그룹화되어 표시되며, 키보드 또는 클릭으로 `/lms-manage/users/[id]` · `/lms-manage/courses/[id]` · `/lms-manage/orders/[id]` 상세 페이지로 이동합니다.

| 입력 | 동작 |
|---|---|
| 1자 미만 | 정적 페이지 목록 표시 |
| 2자 이상 | 4개 도메인 실시간 검색 (도메인별 최대 5건) |
| ↑↓ + Enter | 키보드 탐색 |
| Esc | 닫기 |

---

### 4. 운영 일괄 처리

**무엇**: 반복 운영 작업 4종을 자동화했습니다.

| 기능 | 내용 |
|---|---|
| 학기 복제 | 이전 학기의 분반 구성(강좌·튜터·정원)을 신규 학기에 일괄 복사. 수강 기록 미복사 · 분반 0건 학기에서만 노출 |
| 강좌 복제 | 강좌 메타·차시·시험·문항·과제 전체를 새 강좌로 복사. 수강 기록·제출 이력 미복사. 상태 초안으로 생성 |
| 일괄 액션 | 강좌(게시·마감) / 회원(정지·활성화) 체크박스 선택 후 일괄 처리. 본인 계정 정지 차단 |
| 비밀번호 재설정 | 운영자 화면에서 학습자 임시 비밀번호(16자) 즉시 발급. 기존 세션 전부 만료. 발급 후 1회만 표시 |

**왜**: 학기마다 동일한 분반을 수동 재등록해야 했고, 강좌 구성을 처음부터 재작성하는 반복이 있었습니다. 비밀번호 분실 콜 응대 시 외부 수단(DB 직접 수정)에 의존해야 했습니다.

**결과**: 학기 분반 1클릭 복제로 반복 등록 제거. 강좌 복제 후 수정 시작으로 신규 강좌 준비 시간 단축. 임시 비밀번호 즉시 발급으로 콜 응대 자체 해결.

## 📋 결과 매트릭스

| 도구 | 해소한 문제 | 백엔드 변경 | 프론트엔드 변경 |
|---|---|---|---|
| 학습자 360 뷰 | 회원 정보 5개 메뉴 분산 | KPI 집계 엔드포인트 신규 + 기존 API 필터 확장 | 6탭 컴포넌트 신규 |
| 강좌 운영 허브 | 강좌 운영 동선 분산 | 강좌 KPI 엔드포인트 신규 + 기존 API 필터 확장 | 4탭 컴포넌트 신규 |
| 글로벌 검색 | Command Palette 정적 전용 | 4도메인 통합 검색 엔드포인트 신규 | CommandPalette 확장 + 주문 상세 신규 |
| 학기 복제 | 매 학기 분반 수동 재등록 | 복제 엔드포인트 신규 (DB 트랜잭션) | 분반 복사 버튼·다이얼로그 |
| 강좌 복제 | 강좌 구성 수동 재작성 | 복제 서비스 신규 (DB 트랜잭션) | 복제 버튼·제목 입력 다이얼로그 |
| 일괄 액션 | 개별 행 액션 반복 | 강좌·회원 일괄 처리 엔드포인트 신규 | BulkActionBar + 체크박스 선택 |
| 비밀번호 재설정 | 외부 수단 의존 | 임시 비밀번호 발급 엔드포인트 신규 | 확인 다이얼로그 + 1회 표시 |
