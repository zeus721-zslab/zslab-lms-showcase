# 학점은행 평가인정 정합 트랙

## 개요

학점은행제 평가인정 기관은 NILE이 정한 운영 기준을 충족해야 학점 인정 자격이 유지됩니다. 기준 위반 시 즉시 결격 처리되며, 대표적인 결격 사유로는 학기 일정 미통제·출석 미관리·이의신청 절차 미비가 있습니다. 이 트랙은 해당 결격 사유 세 가지를 수동 운영 의존 없이 시스템이 자동으로 통제하도록 설계합니다.

---

## 정책 근거 (NILE IV.4)

| 기준 항목 | 요구 사항 | 미충족 시 |
|---|---|---|
| IV.4.가 — 학기 일정 관리 | 강좌 운영 기간을 명확히 설정하고 기간 외 응시를 차단 | 학사 관리 부실로 결격 |
| IV.4.나 — 출석 80% 이상 | 학습자의 강좌 출석률이 80% 이상이어야 학점 인정 | 출석 미충족 학점 불인정 |
| IV.4.다 — 이의신청 14일 | 성적 확정 후 14일 이상 이의신청 기간 보장 | 절차 미비로 결격 |

---

## 구현 완료

### 1. 학기 일정 통제 (NILE IV.4.가)

**무엇** — 학기를 네 단계(모집·학습·이의신청·종료)로 파생하여, 학점 시험(중간·기말)의 응시 가능 여부를 학기 단계 기준으로 자동 판정합니다. 학습 기간 외에는 응시가 차단되고, 결과 열람은 허용됩니다. 자격증 시험(모의·본시험)·퀴즈는 학기 일정과 별개로 운영됩니다.

**왜** — NILE 기준상 강좌 운영 기간과 시험 응시 기간이 불일치하면 학사 관리 부실로 평가인정 결격 처리됩니다. 기존에는 운영자가 수동으로 응시 기간을 관리하여 실수 발생 가능성이 있었습니다.

**결과**

| 검증 항목 | 결과 |
|---|---|
| 학습 기간 내 응시 | 정상 허용 |
| 학습 기간 외 응시 차단 | 차단 확인 |
| 이의신청 기간 열람 | 허용 (응시만 차단) |
| 종료 후 결과 공개 | 단계 전환 시 자동 적용 |

---

### 2. 출석 자동 산정 (NILE IV.4.나)

**무엇** — 학습자가 강좌 콘텐츠를 기준 완료율(95%) 이상 수강하면 해당 주차 출석으로 자동 처리합니다. 주차 기준은 콘텐츠 구성과 분리된 별도 주차 단위로 관리하여 NILE 심사 자료로 활용 가능합니다. 총 출석률이 80% 미만이면 학점 인정이 자동 차단됩니다.

**왜** — NILE 기준상 출석 80% 미달 학습자는 학점 인정이 불가합니다. 기존 수동 입력 방식은 담당자 실수·부정 위험이 있어 자동화가 필요했습니다.

**결과**

| 검증 항목 | 결과 |
|---|---|
| 진도 완료 → 출석 자동 처리 | 라이브 확인 |
| 2주 인정 구간 적용 | 정상 동작 |
| 출석률 80% 미달 → 학점 자동 차단 | 차단 확인 |
| 수동 출석 입력 항목 | 자동 산정으로 대체·제거 |

---

### 3. 이의신청 워크플로우 (NILE IV.4.다)

**무엇** — 학점 시험(중간·기말)에 한해 이의신청 비동기 티켓 워크플로우(신청→검토→답변)를 제공합니다. 자격증 시험은 이 경로를 전혀 사용할 수 없도록 5개 지점에서 격리합니다. 이의신청 가능 기간은 학기 이의신청 단계, 또는 학기 정보 없는 상시형 강좌의 경우 시험 종료 후 14일 구간으로 자동 산정합니다.

**왜** — NILE 기준은 성적 이의신청 절차를 14일 이상 보장할 것을 요구합니다. 이의신청 시스템이 없거나 기간이 부족하면 평가인정 결격 사유가 됩니다. 자격증 시험과 학점 시험은 정책 체계가 다르므로 반드시 격리해야 합니다.

**결과**

| 검증 항목 | 결과 |
|---|---|
| 학점 시험 정상 신청 | 201 생성 확인 |
| 자격증 시험 신청 시도 | 차단 확인 (격리 보장) |
| 중복 신청 차단 | 차단 확인 |
| 운영자 검토·답변 | 정상 동작 |
| 종결 후 재답변 시도 | 차단 확인 |
| 라이브 전체 케이스 | 11건 통과 |

이의신청 상태 흐름: **신청 → 검토 중 → 수용 / 반려** (1회 종결, 재신청 불가)

---

### 4. 시험 유형 잠금

**무엇** — 시험 유형(퀴즈·학점 중간·학점 기말·자격증 모의·자격증 본시험)을 데이터베이스·백엔드·API 요청·프론트엔드 상수 네 개 층위에서 동시에 잠급니다. 정의된 유형 외의 값은 어느 경로로도 진입하지 못합니다.

**왜** — 기존 문자열 무제약 상태에서는 개발자 오타·운영자 직접 입력 사고로 잘못된 유형이 삽입될 수 있었습니다. 유형이 오염되면 학점/자격증 시험 정책 로직 전체가 오작동합니다.

**결과**

| 검증 항목 | 결과 |
|---|---|
| 정의된 유형 생성 | 정상 허용 |
| 정의되지 않은 유형 API 요청 | 422 차단 확인 |
| 데이터베이스 직접 오입력 | SQLSTATE 예외 확인 |
| 전체 기존 시험 데이터 회귀 | 이상 없음 |
