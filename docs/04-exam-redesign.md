# 시험 도메인 재설계

## 배경과 문제

기존 구조에서는 시험지와 문항이 같은 테이블에 직접 종속되어 있었습니다. 이 구조에서 문항을 삭제하거나 수정하면 이미 완료된 응시 기록이 파괴되거나 채점 결과가 소급 변조되는 문제가 발생했습니다. 또한 강좌 간 문항 재사용이 불가능하여 운영자가 매 강좌마다 문항을 중복 등록해야 했습니다.

재설계 목표는 세 가지입니다.

1. 문제은행을 독립 마스터 레이어로 분리하여 시험지·응시 기록과 생명주기를 격리
2. 응시 시작 시점에 문항 상태를 캡처(스냅샷)하여 이후 변경과 무관하게 채점 재현성 보장
3. 시험지 복제 시 문항 독립 복사 대신 마스터 참조를 유지하여 문제은행 중복 축적 방지

---

## 핵심 설계 결정

### 1. 문제은행 분리

문항을 시험지와 독립된 마스터 레이어에 보관합니다. 시험지는 문항을 "소유"하지 않고 "차용"합니다. 마스터 문항이 수정되어도 과거 응시 기록에 영향이 없으며, 동일 문항을 여러 시험지에서 참조할 수 있습니다.

```
문제은행 (마스터)
    ↓ 차용
시험지  ←→  출제 규칙 (고정 지정 / 랜덤 규칙)
    ↓ 스냅샷
응시 기록 (응시 시작 시점 문항 본문 캡처)
```

### 2. 스냅샷 기반 응시

학습자가 응시를 시작하는 시점에 해당 시험지의 모든 문항 본문을 별도 테이블에 복사합니다. 이후 마스터 문항이 수정되거나 삭제되어도 응시 기록은 스냅샷을 기준으로 채점됩니다. 스냅샷이 없는 레거시 응시 기록은 기존 경로로 폴백하여 채점 연속성을 유지합니다.

### 3. 복제 시 공유 참조 유지

시험지를 복제할 때 문항을 독립 복사하지 않고 마스터 참조를 그대로 인계합니다. 강좌 간 복제를 지원하며, 고정 출제·랜덤 출제 두 모드 모두 복제 후에도 구성 방식을 유지합니다.

---

## 도메인별 구현 결과

### 데이터 모델 재설계

기존 단일 테이블 구조를 네 개 레이어(문제은행 마스터·시험지 고정 출제 목록·랜덤 출제 규칙·응시 스냅샷)로 분리했습니다. 기존 문항 데이터는 무결성을 유지한 채 마스터 레이어로 이관했습니다.

| 항목 | 내용 |
|---|---|
| 신규 테이블 | 4개 |
| 마이그레이션 | 6개 |
| 기존 데이터 이관 | 무결성 유지 확인 |
| 회귀 케이스 | 20건 통과 |

---

### 문제은행 관리

운영자·교수가 문항을 중앙에서 등록·수정·삭제하고, 시험지에 차용합니다. 난이도 분류와 CSV 일괄 등록을 지원하여 대규모 문항 도입이 가능합니다.

| 기능 | 내용 |
|---|---|
| 문항 CRUD | 단건 등록·수정·삭제 |
| CSV 일괄 등록 | 다수 문항 한 번에 import |
| 난이도 분류 | 상·중·하 태깅 |
| 강좌 간 공유 | 마스터 참조로 재사용 |

---

### 시험지 구성

시험지 구성 방식은 고정 모드와 랜덤 모드 두 가지입니다.

**고정 모드** — 출제할 문항을 개별 지정하고 드래그 앤 드롭으로 순서를 조정합니다. 다른 시험지에서 사용 중인 문항을 차용하는 기능을 포함합니다.

**랜덤 모드** — 난이도·유형 조건을 설정하면 응시마다 문항 풀에서 무작위 추출합니다. 풀 규모가 출제 수의 4배 미만일 경우 시각 경고를 표시하고, 절대 부족(출제 수 이하)이면 차단합니다. 4배 기준은 NILE 평가인정 심사에서 문항 노출 방지를 위해 권장하는 최소 풀 규모 관행에 따릅니다.

| 항목 | 고정 모드 | 랜덤 모드 |
|---|---|---|
| 출제 방식 | 문항 직접 지정 | 조건 기반 자동 추출 |
| 순서 제어 | 드래그 앤 드롭 | — |
| 풀 검증 | — | 4배수 시각화·부족 시 차단 |
| 미리보기 | ✅ | ✅ |

---

### 응시 스냅샷·채점 이중 경로

응시 시작 시 문항 본문이 스냅샷으로 고정됩니다. 채점 시 스냅샷 우선, 스냅샷 없는 레거시 기록은 기존 경로로 폴백합니다.

| 항목 | 내용 |
|---|---|
| 스냅샷 생성 시점 | 응시 시작 직후 |
| 채점 우선순위 | 스냅샷 → 레거시 폴백 |
| 마스터 문항 수정 후 기존 응시 | 스냅샷 기준 채점 유지 |
| 레거시 응시 기록 | 기존 경로 유지 (하위 호환) |

---

### 학습자 응시 UI

**응시 흐름 개선**

- 시험 유형·최대 응시 횟수·잔여 횟수를 응시 전 화면에 명시
- 응시 중 서버 측 임시 저장으로 브라우저 종료·재접속 후 이어서 응시 가능

**결과 리뷰 — 시험 유형·학기 단계별 차등 공개**

공개 범위는 시험 유형과 학기 단계를 조합하여 자동 결정됩니다.

| 시험 유형 | 정답 공개 | 내 답안 공개 | 해설 공개 |
|---|---|---|---|
| 퀴즈 | 즉시 | 즉시 | 즉시 |
| 학점 시험 (중간·기말) | 이의신청 기간 종료 후 | 이의신청 기간 종료 후 | 이의신청 기간 종료 후 |
| 자격증 모의시험 | 즉시 | 즉시 | 즉시 |
| 자격증 본시험 | 정답만 공개 | 비공개 (영구) | 비공개 (영구) |

학점 시험 이의신청 기간 중 정답을 공개하면 이의신청 의미가 훼손되므로, 이의신청 단계 종료 후 공개로 설계했습니다. 자격증 본시험은 실제 자격증 기관 운영 방식에 준하여 본인 답안을 공개하지 않습니다.

---

### 시험지 복제

운영자·교수가 기존 시험지를 복제하여 신규 강좌에 적용할 수 있습니다. 고정·랜덤 두 모드 모두 복제 후 구성 방식이 유지되며, 동일 강좌 내 복제와 다른 강좌로의 복제를 지원합니다.

| 항목 | 내용 |
|---|---|
| 고정 모드 복제 | 문항 지정 목록 그대로 인계 (마스터 참조 유지) |
| 랜덤 모드 복제 | 조건 규칙 그대로 인계 |
| 다른 강좌 복제 | 지원 |
| 검증 케이스 | 11건 통과 |

## 결과 매트릭스

| 도메인 영역 | 무엇 | 왜 | 결과 |
|---|---|---|---|
| 데이터 모델 | 문항·시험지·응시 레이어 분리 | 기존 종속 구조 → 삭제 시 응시 기록 파괴 | 4테이블 신규·기존 데이터 이관 완료 |
| 스냅샷 | 응시 시작 시 문항 본문 캡처 | 사후 문항 수정 시 채점 변조 방지 | 스냅샷 기준 채점·레거시 폴백 병행 |
| 랜덤 출제 4배수 검증 | 풀 규모 미달 시 경고·차단 | NILE 평가인정 심사 권장 기준 | 3단계 시각화·절대 부족 시 API 차단 |
| 차등 공개 | type×학기 단계 조합으로 공개 범위 결정 | 이의신청 중 정답 공개 시 절차 훼손 | 4유형×단계 매트릭스 구현·11케이스 통과 |
| 시험지 복제 | 마스터 참조 유지 복제 | 독립 복사 시 문제은행 중복 축적 | 고정·랜덤 분기·강좌 간 복제 지원 |
| 응시 임시 저장 | 서버 측 진행 상태 임시 저장 | 브라우저 종료·재접속 시 이어서 응시 | 임시 저장·복원 라이브 확인 |
